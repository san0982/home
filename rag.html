RAG 檢索策略演進

精準度 vs. 實現複雜度 綜合分析

核心權衡：投資與回報

「樸素 RAG」(Naive RAG) 依賴基礎的向量相似度檢索，雖然實現簡單，但在面對複雜查詢時精準度有限。為了突破此瓶頸，業界發展出多種優化路徑。此圖表分析了不同檢索策略在「檢索精準度」與「實現複雜度」兩個關鍵維度上的權衡關係。複雜度不僅代表技術門檻，也隱含著更高的開發、維護與營運成本 (OpEx) 或資本支出 (CapEx)。

檢索方法深度解析

🚀

樸素 RAG (Naive RAG)

基礎的密集檢索（Dense Retrieval），僅使用向量相似度。

優點：實現簡單、快速、成本最低。

缺點：資訊損失、面對複雜查詢時精準度低。

⚖️

混合式檢索 (Hybrid)

結合稀疏檢索（如 BM25）與密集檢索（向量）的優點。

優點：平衡關鍵字與語義理解，精準度顯著提升。

缺點：需要管理兩種索引，架構稍顯複雜。

📊

Reranking (重排)

在檢索後，使用更強大的模型對 Top-K 結果進行重新排序。

優點：目前提高精準度的 SOTA 標準實踐。

缺點：增加額外延遲；面臨 API (OpEx) 或自架 (CapEx) 的成本選擇。

🤖

Agentic RAG

透過自主代理（Agents）進行規劃、反思與多步檢索。

優點：適應非結構化、動態且需多跳躍推理的任務。

缺點：高延遲、高 Token 消耗，邏輯複雜。

🕸️

KRAG (知識圖譜 RAG)

從結構化的知識圖譜中檢索，提供確定性知識。

優點：邏輯嚴謹性高，利於全域理解與複雜推理。

缺點：知識圖譜的建構與維護成本極高。

🧠

微調 / 新架構 (Finetuning)

微調 Embedding 模型或採用 MemoRAG 等新架構。

優點：可達到最高的性能上限，上下文感知能力強。

缺點：最高的成本和複雜性，需要大量標註數據。

RAG 優化成本金字塔

RAG 的優化決策可視為一個成本金字塔。應從基礎層開始實施，並根據業務需求和預算，策略性地選擇是否向頂層投資，以換取更高的精準度。

頂層 (按需選擇)

微調 / 新架構 (MemoRAG)

(極高成本，最高性能)

中間層 (策略選擇)

Reranking / Agentic RAG / KRAG

(高成本，高精準度)
